Employee Census Data Manager: User Guide

Welcome to the Employee Census Data Manager! This tool standardizes your client's raw census file so that the data can be stored securely and used reliably by all compliance calculators (like ADP/ACP and  Testing).

🚀 Step 1: Upload Your CSV File

The first step is to locate and upload the census file provided by your client.

  1. Navigate to the Employee Census Data Manager under the Administration & Settings group on the main dashboard.
  2. Scroll to Step 1: Upload CSV File.
  3. Click the Choose File button and select the employee census data file (it must be a standard .csv file).

Once the file is selected, the system automatically parses the headers and moves you to Step 2.

📝 Step 2: Map Your Columns

Because every payroll system uses different column names (e.g., "Pay", "Comp", "W-2 Compensation"), this crucial step ensures your custom headers are matched to the software's Standard Required Fields.

The tool will display a list of all columns found in your uploaded CSV on the left. On the right, you must select the corresponding standard field from the dropdown menu.

Mandatory Fields

You MUST map the following four fields for the data to be considered valid for compliance testing:

Your CSV Header

Software Standard Field

Example Mapping

ID_NUM

Employee ID (Req)

(For unique identification)

DOB_FMT

Date of Birth (YYYY-MM-DD) (Req)

(For age testing/HCE status)

W2_COMP

Annual Compensation ($) (Req)

(For contribution limits/testing)

ACCOUNT_BAL

Vested Account Balance ($) (Req)

(For Top-Heavy testing)

Handling Unnecessary Columns

If a CSV column is not relevant to compliance testing (e.g., HIRE_DATE, PLAN_STATUS), simply leave the dropdown for that field set to -- Ignore Column --.

Once mapping is complete, click the Process & Validate Data button.

💾 Step 3: Process, Validate, and Save

After processing, the tool performs data validation and conversion (e.g., converting text-based compensation into pure numerical values).

Data Validation

  • If any Required Field (listed above) is missing a value for an employee, that employee record will be marked as invalid and will not be included in the final data set.
  • The Status Message at the top of the screen will report the number of records processed and any validation errors found.

Preview and Final Save

  1. Review the Data Preview table at the bottom. This table shows the first 100 valid records in the standardized format.
  2. If the data looks correct, click the Save Processed Data to Plan button.
